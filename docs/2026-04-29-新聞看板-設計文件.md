## 1. 專案目的

打造一個個人化、永遠在線的全球資訊 Dashboard，每天早上 7 點（台灣時間）透過 WhatsApp 推播當日重點，作為一日資訊起點。

**設計核心原則：**
- 「即時更新」≠「完全即時」：實際每 30 分鐘更新一次（已足夠）
- 「全球重點」≠「全部新聞」：每類別精選 Top 2-5 條，避免資訊過載
- 「不點開也有用」：WhatsApp 訊息本身就具備 80% 認知價值
- 「忙的時候可以略過」：通知是提醒，不是命令

---

## 2. 使用者場景

> 每天早上 7 點，WhatsApp 響一聲。鮪魚（使用者）打開訊息，30 秒掃完六大領域重點。
> - 有興趣 → 點 Dashboard 連結看完整內容
> - 沒空 → 已掌握世界大事，繼續做別的事

## 4. 資料來源（按優先順序）

### 4.1 每 30 分鐘抓取

| 類別 | 來源 | 抓取方式 | 備註 |
|------|------|----------|------|
| 🌍 **世界** | Reuters World, BBC Business, CNN Markets | RSS | 英文 |
| 💰 **財經（新聞）** | Reuters Business, 商業周刊 | RSS | 中英混合 |
| 💰 **財經（市場數據）** | Yahoo Finance API | API | 台股加權、美股三大指數、台幣匯率、美元指數 |
| 🪙 **加密（新聞）** | CoinDesk, The Block, ABMedia 鏈新聞, BlockTempo 動區 | RSS | 中英混合 |
| 🪙 **加密（市場數據）** | CoinGecko API, Alternative.me | API | BTC/ETH 等主流幣價、恐慌貪婪指數 |
| 💻 **科技** | TechCrunch, VentureBeat | RSS | 英文 |
| 🎬 **影視** | Hollywood Reporter, Variety | RSS | 英文 |

### 4.2 每週一掃描

| 類別 | 來源 | 抓取方式 | 通知策略 |
|------|------|----------|----------|
| 📚 **機構報告** | Goldman Sachs Reports, WEF, CFR, PwC Taiwan, TAICCA, McKinsey GEI, IMF WEO, Goldman 2026 Outlooks | 網頁爬蟲 + sitemap 比對 | 偵測到「新報告 URL」時 → AI 摘要 → WhatsApp |

> **注意**：機構報告多為 PDF / 會員牆 / 無 RSS。MVP 階段先做「網址比對偵測新報告」，下載與摘要邏輯放在第二階段。

## 5. 三個排程任務

### 5.1 Fetch & Summarize（每 30 分鐘）

1. 並行抓取所有 RSS / API 來源
2. 每筆文章用 hash（URL + 標題）去重，已存在跳過
3. **新文章**送 Claude Haiku 4.5 → 產生 1 句摘要（繁體中文）
4. 寫回 `data/news.json` 和 `data/market.json`
5. Commit & push（觸發 GitHub Pages 重建）

### 5.2 Daily Brief（每天 06:55 TPE）

1. 讀過去 24 小時的 news.json
2. 每類別挑 Top 2 條，排序規則（依序）：
   - (a) 來源權重：每來源於 `config.yml` 預設 1-3 分（例：Reuters=3, BBC=3, BlockTempo=2）
   - (b) 發布時間越新越優先（24 小時內遞減衰減）
   - (c) 同分時取標題較長者（資訊量代理指標）
3. 拉取最新 market 快照
4. 套用 brief template → 輸出純文字
5. 呼叫 CallMeBot HTTP → 推送至鮪魚 WhatsApp

### 5.3 Weekly Reports（每週一 09:00 TPE）

1. 對每個機構網站 fetch 報告列表頁面
2. 與上週存檔的 URL 清單比對
3. 有新報告 → 送 WhatsApp 通知（含報告連結）
4. 第二階段：自動下載 PDF + Claude 摘要

## 7. Dashboard 介面（GitHub Pages）

### 7.1 結構

- **Header**：日期 + 最後更新時間 + 加密 / 股市 ticker
- **Section 1**：六大類別卡片（依優先順序排列）
  - 每卡片：類別圖示 + 標題 + Top 5 條目
  - 每條目：標題（連結原文）+ 來源 + 時間 + AI 摘要
- **Section 2**：機構報告區（顯示最近 4 週新增報告）

### 7.2 技術選型

- **純靜態**：HTML + Tailwind（CDN）+ Vanilla JS
- **資料**：`fetch('./data/news.json')` 從同源讀取
- **自動刷新**：每 5 分鐘 `location.reload()` 或 `setInterval` 重抓 JSON
- **響應式**：手機優先（單欄），桌機 2 欄
- **無框架**：簡單、可維護、不需要 build step

## 8. 錯誤處理

| 失敗情境 | 處理方式 |
|----------|----------|
| 單一 RSS 來源失敗 | log warning、跳過、繼續其他來源 |
| Claude API 失敗或超額 | 該批次推「無摘要版」（只列標題） |
| WhatsApp 推播失敗 | 重試 3 次（間隔 30 秒），全失敗 → 自動建 GitHub Issue |
| GitHub Actions 執行失敗 | Actions 內建通知（會寄 email 到 wenyupan911202@gmail.com） |
| 資料完全為空 | 該分類顯示「📚 今日無更新」而非崩潰 |

## 13. 不在範圍內（Out of Scope）

明確排除以避免範圍蔓延：

- ❌ 多使用者支援（僅鮪魚一人）
- ❌ 行動 App（用瀏覽器即可）
- ❌ 評論 / 收藏 / 分享功能
- ❌ 歷史搜尋介面（資料保留 30 天，更舊的歸檔）
- ❌ Push 推播以外的通知管道（不做 LINE / Email / Telegram）
- ❌ 真正的即時推送（WebSocket / SSE）
